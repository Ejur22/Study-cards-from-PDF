# Анализ Проблем и Исправления

## 🔴 ПРОБЛЕМА 1: Адресная строка не меняется при переходе между страницами

### Диагностика:
- **Причина:** Приложение использовало **state-based routing** вместо React Router
- **Как это работало:** в App.jsx был `const [currentScreen, setCurrentScreen] = useState('main')`
- **Проблема:** Все компоненты рендерились в зависимости от значения `currentScreen`, но URL всегда оставался `/`
- **Результат:** Пользователь не может сохранить ссылку на страницу, при обновлении возвращается на главную

### Решение:
✅ Обновлён App.jsx для использования React Router:
- Заменены `setCurrentScreen()` вызовы на `navigate()`
- Создана структура маршрутов через `<Routes>`с правильными путями:
  - `/` - главная страница
  - `/registration` - регистрация
  - `/login` - вход
  - `/quiz/:groupId` - тест
  - `/results` - результаты
  - `/history` - история обучения
  - `/users` - админка
  - `/*` - 404 страница
- Браузер теперь показывает правильные пути в URL

## 🔴 ПРОБЛЕМА 2: Параметры сортировки видны везде (утечка query params)

### Диагностика:
- **Причина:** HistoryPage обновляет URL с параметрами фильтра:
  ```javascript
  window.history.replaceState({}, '', `${window.location.pathname}?${params}`)
  ```
- **Проблема:** Когда пользователь переходит на другую страницу, параметры остаются в URL
- **Результат:** На главной странице видны `?search=test&sort_by=created_at` даже если это не История

### Решение:
✅ Исправлена логика роутинга:
- React Router автоматически переходит на новый путь `/` без параметров
- Параметры фильтра уже не будут видны на других страницах
- HistoryPage сохраняет параметры только на `/history`

## 🔴 ПРОБЛЕМА 3: При вводе 404 нет ошибки, редирект на главную

### Диагностика:
На самом деле это НЕПРАВИЛЬНО интерпретировалось:
- **Что происходит на бэкенде:**
  - catch-all маршрут `/{full_path:path}` в public.py правильно возвращает 404 HTTP ошибку
  - Это правильное поведение
- **Что происходит на фронтенде (до исправления):**
  - Фронтенд не знал о несуществующих путях, потому что всё рендерилось локально
  - React Router не был использован, поэтому `/random-path` просто показывал главную

### Решение:
✅ Добавлена React Router логика:
- Нестоящих маршрутов обрабатываются через `<Route path="*" />`
- Пользователь видит правильную 404 страницу с сообщением об ошибке
- Кнопка для возврата на главную

---

## 📝 Что было исправлено в коде:

### App.jsx
✅ Заменён state-based routing на React Router:
```jsx
// БЫЛО:
const [currentScreen, setCurrentScreen] = useState('main')
setCurrentScreen('quiz')

// СТАЛО:
const navigate = useNavigate()
navigate('/quiz/:groupId')
```

✅ Добавлены React Routes:
```jsx
<Routes>
  <Route path="/" element={<MainScreen />} />
  <Route path="/login" element={<LoginScreen />} />
  <Route path="/registration" element={<RegistrationScreen />} />
  <Route path="/quiz/:groupId" element={<QuizScreen />} />
  <Route path="/results" element={<ResultsScreen />} />
  <Route path="/history" element={<HistoryPage />} />
  <Route path="/users" element={<UsersPage />} />
  <Route path="*" element={<404 Page />} />
</Routes>
```

### HistoryPage.jsx
✅ Комментарий добавлен о том, что параметры теперь локальны для страницы

### dictionary_service.py
✅ Полностью переписан с реальным Яндекс.Словарь API:
- Используется `https://dictionary.yandex.net/api/v1/dicservice.json/lookup`
- Добавлена функция `_fetch_from_yandex_api()` которая парсит API ответ
- Возвращаемый формат совпадает с требуемым
